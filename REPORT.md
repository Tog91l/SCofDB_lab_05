## 2. Демонстрация неконсистентности (намеренно сломанный сценарий)
_TODO: Опишите шаги и результаты stale cache._

Минимум:
1. Прогрев кэша.
2. Изменение заказа в БД без инвалидации.
3. Повторный запрос из кэша.
4. Чем stale ответ отличается от данных в БД.

1. выполняем запрос, данные сохраняются в Redis
2. Обновляем бд в sql без инвалидации
3. опять запрашиваем из кэша
4. данные остаются старыми stale
Пользователь получит старые данные , так как не было инвалидации
Первое значение total_amount: 100.0
Второе значение total_amount: 100.0
Причина: кэш не был инвалидирован после изменения БД
